## SECTION 3: ROOT CAUSE ANALYSIS - FLOORPLANNING FAILURE

### What Happened

When floorplanning tried to run via Docker, it got:
```
[ERROR PPL-0021] Horizontal routing tracks not found for layer met3.
```

### Root Causes Identified

1. **PDK Mounting Issue**:
   - TCL script references `/pdk/sky130A/libs.ref/sky130_fd_sc_hd/...`
   - PDK needs proper mount from host to `/pdk` in container
   - Current attempt to mount may have Windows path issues

2. **Liberty File Availability**:
   - Script needs `sky130_fd_sc_hd__tt_025C_1v80.lib`
   - File must exist at `/pdk/sky130A/libs.ref/sky130_fd_sc_hd/lib/` inside container

3. **Missing PDK Data**:
   - Routing layers (met3, met4, etc.) not available
   - These Define metal layer characteristics

### How to Fix (Prioritized)

#### Option A: Use Docker Internal PDK (Recommended for quick fix)
The OpenLane Docker image may have a built-in PDK. Modify docker_manager.py:
```python
# Try to use internal PDK first if mounted PDK fails
if pdk_root_failed:
    env_args += ["-e", "PDK_ROOT=/opt/pdks"]  # or wherever it's in the image
```

#### Option B: Proper Windows-to-Docker PDK Mount (Robust)
The volare download is working (detected at C:\pdk). Fix the mount path:
```python
# In docker_manager.py _docker_run_with_work()
if self.pdk_root:
    # Convert Windows path properly for Docker
    pdk_path = Path(self.pdk_root)
    if pdk_path.name == "sky130A":
        pdk_path = pdk_path.parent
    pdk_docker_path = str(pdk_path).replace("\\", "/")
    # Verify it's accessible
    mounts.extend(["-v", f"{pdk_docker_path}:/pdk:z"])  # :z for SELinux compatibility
```

#### Option C: Debug Current Mount
Add logging to see what's being mounted:
```python
self.logger.info(f"Mounting {self.pdk_root} to /pdk")
self.logger.info(f"PDK exists at {self.pdk_root}?  {Path(self.pdk_root).exists()}")
if (Path(self.pdk_root) / "sky130A").exists():
    self.logger.info("✓ sky130A found in mount")
```
